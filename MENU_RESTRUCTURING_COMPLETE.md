## 🔧 기술 구현

### 1. Alpine.js 통합
```html
<!-- CDN 추가 -->
<script defer src="https://cdn.jsdelivr.net/npm/@alpinejs/collapse@3.x.x/dist/cdn.min.js"></script>
<script defer src="https://cdn.jsdelivr.net/npm/alpinejs@3.x.x/dist/cdn.min.js"></script>
```

### 2. 상태 관리 패턴
```javascript
x-data="{
    category1: true,   // 기본 열림
    category2: false,
    category3: false
}"
```

### 3. 접기/펼치기 구조
```html
<button @click="category = !category">
    <span>카테고리명</span>
    <svg :class="{ 'rotate-180': category }">
        <!-- 화살표 -->
    </svg>
</button>
<div x-show="category" x-collapse>
    <!-- 하위 메뉴 -->
</div>
```

### 4. 색상 시스템
- 🔵 Blue: 수업/학사 관련
- 🟢 Green: 학생/학습 관련
- 🟣 Purple: 과제/첨삭 관련
- 🟡 Yellow: 소통 관련
- 🟠 Orange: 재무 관련
- 💗 Pink: 콘텐츠 관련

## 🎓 주요 학습 포인트

### 1. 정보 아키텍처
- **인지 심리학**: 7±2 법칙 (단기 기억 한계)
- **정보 청킹**: 관련 정보를 그룹으로 묶기
- **계층 구조**: 2단계 (카테고리 → 하위 메뉴)

### 2. 일관성의 중요성
- 모든 포털에서 동일한 패턴 사용
- Alpine.js, 색상 시스템, 호버 효과 일관 적용
- 사용자가 한 포털에서 배운 패턴을 다른 포털에 적용 가능

### 3. 사용자 중심 설계
- 각 포털의 주요 사용자 니즈 파악
- 가장 자주 사용하는 카테고리를 기본 열림으로
- 단일 메뉴와 다중 메뉴의 적절한 혼용

### 4. 점진적 개선
- 한 번에 모든 포털을 변경하지 않고 단계적 진행
- 각 단계마다 테스트 및 검증
- 이전 단계의 패턴을 다음 단계에 적용
