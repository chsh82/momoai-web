# 🎯 MOMOAI v4.0 메뉴 구조 재구성 제안서

## 📊 현재 구조 분석

### 🔴 현재 구조의 문제점

1. **평면적 구조**: 모든 메뉴가 1단계로 나열되어 있어 찾기 어려움
2. **논리적 그룹 부재**: 관련 기능들이 흩어져 있음
3. **메뉴 수 과다**: 관리자 메뉴만 20개 이상으로 스크롤 필요
4. **일관성 부족**: 포털별 메뉴 구조가 다름
5. **중복 아이콘**: 같은 이모지(📚, 📊 등)가 여러 메뉴에 사용됨

---

## ✅ 제안하는 새로운 계층 구조

### 설계 원칙

1. **2단계 계층**: 상위 카테고리 → 하위 메뉴
2. **기능별 그룹화**: 관련 기능을 묶어 찾기 쉽게
3. **사용 빈도 고려**: 자주 쓰는 기능은 상단에 배치
4. **일관성**: 모든 포털에서 유사한 구조 유지
5. **확장성**: 새 기능 추가가 쉬운 구조

## 🎨 UI/UX 개선 사항

### 1. 접을 수 있는 메뉴 (Collapsible Menu)

```
📚 학사 관리 ▼                    [열림]
  ├─ 수업 관리
  ├─ 전체 수업현황
  └─ 출석 체크

📚 학사 관리 ▶                    [닫힘]
```

### 2. 아이콘 색상 코드

- 🔵 **학사/수업**: 파란색 계열
- 🟢 **학습/콘텐츠**: 초록색 계열
- 🟡 **소통**: 노란색 계열
- 🟠 **재무**: 주황색 계열
- 🔴 **관리**: 빨간색 계열

### 3. 배지 표시

```
📢 공지사항          [3]  ← 읽지 않은 개수
✍️ 과제 관리         [5]  ← 대기 중인 첨삭
```

## 📋 구현 우선순위

### Phase 1: 관리자 포털 (1주)
- 상위 카테고리 구조 생성
- 기존 메뉴를 카테고리별로 재배치
- 접기/펼치기 기능 구현

### Phase 2: 강사/학생/학부모 포털 (1주)
- 동일한 구조 적용
- 포털별 커스터마이징

### Phase 3: UI/UX 개선 (3일)
- 아이콘 색상 적용
- 배지 시스템
- 애니메이션 효과

### Phase 4: 테스트 & 피드백 (2일)
- 사용성 테스트
- 피드백 반영
